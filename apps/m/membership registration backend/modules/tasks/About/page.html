What is a micro app

"A Micro-app is a super-specialized application designed to perform one task or use case with the only objective of doing it well." -- From Wikipedia

Cost of cloud servers for running a micro app


  • Web Page Server: using GitHub Pages free services. More information.
  • API server (AWS Micro -- 2 vCPU, 1 GB DAM, 20 GB DISK + comunication): AUD $10/Month plus
  • Database and file server (AWS Micro -- 2 vCPU, 1 GB RAM, 30 GB DISK): AUD $10/Month
Roughly total $25/month
Some other cost such as domain, SSL etc may be incurred.


Authentication

We use Microsoft, Google, Facebook or Github authentication services and use a person's email address to identify the login. When a user send a request to the server, the email address will be used as unique id for permissions control.


Permissions Control

We use a configuration file to control the permissions. An exampe is as follow.

{
    "groups":{
        "global_admin":["admin@company365.com","supert@company365.com"],
        "it":["tom@company365.com",expert@gmail.com],
        "finance":["david@gmail.com"]
    },
    "permission_table":{
        "/finance_read":    {"azure_groups":[], "groups":["finance"], "users":[]   },
        "student_full":     {"azure_groups":[], "groups":["it"], "users":[]   },
        "student_read":     {"azure_groups":["all_staff"], "groups":[], "users":["jimmy@gmail.com","anna@gmail.com"]   }
    }
}
An administrator can use a text editor to edit the configuration file.

Notes:
"/finance_read" means reading the files in the folder /finance.
"student_read" means reading the data in database table called "student".
